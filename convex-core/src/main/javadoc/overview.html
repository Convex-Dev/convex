Convex

Convex is a decentralised lattice network and execution platform. It realises the vision of a Stateful Internet — a global, public network that securely hosts, executes, and persists both code and data, powered by lattice technology and Convergent Proof of Stake (CPoS) consensus.

These Javadocs cover the reference Java implementation of the Convex protocol, published as artifacts under the world.convex group on Maven Central.

Module Map

Convex is a multi-module Maven project. The most commonly referenced artifacts are:

  • convex-core — CVM, CAD3 data structures, CPoS consensus, Etch storage
    • {@link convex.core} — core package and module entry point
    • {@link convex.core.data} — immutable persistent data structures (ACell hierarchy)
    • {@link convex.core.cvm} — Convex Virtual Machine and state transition function
    • {@link convex.core.lang} — Convex Lisp compiler, reader, and runtime
    • {@link convex.core.cpos} — Convergent Proof of Stake consensus
    • {@link convex.core.crypto} — Ed25519 signatures and SHA-256 hashing
    • {@link convex.core.store} — pluggable content-addressed storage
    • {@link convex.etch} — high-performance persistent Etch storage engine
    • {@link convex.lattice} — lattice abstractions underpinning DLFS, KV stores, and more
    • convex.auth — DID, JWT, and UCAN authorisation primitives
  • convex-peer — peer server, binary protocol, P2P networking
  • convex-restapi — HTTP REST API server
  • convex-java — Java client library for connecting to Convex peers
  • convex-cli — command-line tooling and scripting
  • convex-gui — Swing desktop application

Design Principles

  • Immutable data — every CVM value is an immutable {@link convex.core.data.ACell} with a canonical CAD3 encoding and content-addressed hash.
  • Deterministic execution — the CVM is a pure, deterministic state transition function; every peer computes identical results given the same inputs.
  • Lattice technology — state converges through commutative, associative, idempotent merges rather than a linear chain, enabling high throughput and flexible replication topologies.
  • Global state model — a single public state tree hosts self-sovereign accounts (e.g. #1337), on-chain actors, coins, and arbitrary application data.
  • Security by default — peer and REST surfaces are hardened against malicious messages; lattice nodes are private unless explicitly shared.
